Description

This is a project for client, who wanted to have dynamic page for his own business. I decided to created it in Wordpress, to have ability to add dynamic new lessons, events with text, photos and videos.

I build own theme which contain:

  1. 1. Header with Logo and menu (mobile menu)
  2. 2. Front Page with sections:
    • Section gallery slider. (ability to add / change 4 images)
    • Section welcome with video.
    • Section courses (by choosing 4 pages).
    • Section events. Events added by custom post type events.
    • Section message (by choosing one page)
  3. 3. Fourth Custom Archive Pages for each different group. Content is displayed for people who are logged and have added role. 
  4. 4. One Page with calendar. In calendar, events are automatically added from event post type. When user want to add event need to write date of event in advanced custom field. Also can add color of event, type, and short description. For this I added new router, and used json to get data and added into evocalendar function. The calendar which I used it is from open source from github. I customized it for this website needs. 
  5. 5.Footer - User can add 4 widgets to footer.
  6. 6. Also theme has: 
    • Two Sidebar
    • Extra widgets for events, which can be added to sidebar. Widget display just 3 earliest planned events. 

Website is still in build. This version is for test and for client view.

Opis

Projekt utworzony dla klienta, który chciał mieć dynamiczną stronę dla własnych celów. Stronę utworzyłam w Wordpresie, w celu zapewnienia możliwości łatwego dodawania lekcji, wydarzeń z tekstem, zdjęciami, plikami video.

Zbudowałam motyw, który zawiera.

  1. 1. Header z logo oraz menu
  2. 2. Strona główna z sekcjami:
    • Sekcja – slider ze zdjęciami (możliwość dodawania zmieniania 4 zdjęć).
    • Sekcja – powitalna z plikiem wideo.
    • Sekcja możliwe kursy (możliwość wybrania czterech stron, do sekcji zostanie dodane video).
    • Sekcja – wydarzenia. Wydarzenia są dodawane poprzez własny typ postów (event)
    • Sekcja wiadomość – informacja o stronie (wybranie strony )
  3. 3. Cztery typy własnych stron (archive pages). Każda strona przeznaczona jest dla osobnej grupy docelowej. Informacje stron są wyświetlane dla osób które są zalogowane i mają określoną rolę użytkownika do wyświetlania informacji.
  4. 4. Strona z kalendarzem. Do kalendarza dodawane są wydarzenia z własnego typu postów ,, event”. Jeśli użytkownik chce dodać wydarzenie do kalendarza musi dodać datę w ACF (advance custom fields) Może również dodać opis, color wydarzenia, typ wydarzenia. Do tego celu utworzyłam custom url route w celu wykorzystania json dla kalendarza evo-calendar. Kalendarz pobrałam z open source na githubie. Kalendarz dostosowałam do potrzeb strony.
  5. 5. Stopka, użytkownik może dodać cztery widgety do stopki.
  6. 6. Sidebar - również strona posiada dwa sidebar’y, oraz dodatkowy widget dla wydarzeń (wyświetlanie 3 najwcześniejszych wydarzeń)

Strona jest wciąż w budowie. Obcnie jest to wersja testowa do wglądu dla klienta.
